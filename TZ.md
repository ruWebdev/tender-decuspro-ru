Техническое задание на разработку тендерной площадки (для нужд единственного заказчика)


Тендеры происходят не быстро, поставщики могут заходить в течении длительного времени и перебивать цены друг друга.


Модуль заказчика:
• Регистрация и вход (email + пароль)
• Создание новой закупки:
oНазвание закупки(Тема)
oОписание/комментарий (информация для поставщиков)
oСкрытый комментарий (информация для заказчика)
oПозиции товаров/услуг (название, количество) в построчной форме
oДата Актуально до


oАвто заполнение построчной формы через DeepSeek API 
(вставляем список товаров текстом в окно (описание - кол-во ), deepseek на основе этих данных самостоятельно заполняет форму построчно)


• Просмотр списка созданных закупок (тема, скрытый комментарий, кол-во поданных предложений, сумма лучшего предложения)
• Выбор победителя (при выборе победителя, остальные видят что проиграли, на странице тендера появляется информация о том что конкурс завершен и победитель выбран)


Модуль поставщика:
• Регистрация с информацией о компании:
oНазвание компании
oКонтактные данные
oЗагрузка документов (PDF/фото)
• Просмотр списка активных закупок на главной
• Подача/обновление предложения (цена на каждую позицию + комментарий)
• Просмотр текущих цен (информация о поставщиках скрыта)
• Просмотр статуса своих заявок
•Получение уведомления на email если перебили твою цену


Общее:
• Главная страница с лентой закупок (тема, создано дата, актуально до)
•Возможность редактировать информационный блок на главной странице
• Уведомления на email о новых действиях
• Три языка для пользовательской части: русский – основной. Перевод на Англ и Китайский через API.
•Возможность выбора языка на любой странице 




Техническая реализация
Backend: Laravel (API, Sanctum, Queues, Notifications, File storage)
Frontend: Vue 3 + PrimeVue/Quasar + Pinia
База данных: PostgreSQL + Eloquent ORM
AI интеграция: DeepSeek API через Laravel HTTP Client + Queues
Особенности: Встроенная аутентификация, уведомления на email, мультиязычность, загрузка файлов "из коробки"
